# PoCポーティングチェックリスト

## 目的

`ref/` 配下のPoC結果をD2Dの仕様書・設計書へ取り込む際に、PoCの有効な設計知見を落とさず、D2Dの全体方針へ写像するためのチェックリストである。

対象はWordに限らず、Excel、PowerPoint、PDF、Visio等の抽出PoC全般とする。

## なぜなぜ分析

| なぜ | 原因 |
| --- | --- |
| なぜPoCの重要な考え方が一部落ちたか | ポーティングガイドのUI/API寄り情報を先に反映し、詳細設計の責務分解を十分に設計へ写像しなかったため |
| なぜ詳細設計の責務分解が落ちたか | PoC成果物を「機能一覧」として読んでしまい、データ構造、パーサ、ジェネレータ、検証の実装契約として棚卸ししなかったため |
| なぜSRSに実装寄りの曖昧語が入ったか | SRSとSDDの役割分担を明確にせず、実装名を要求仕様へそのまま持ち込んだため |
| なぜ保存先の重複が生じたか | PoCの出力ファイルを、D2D上の正本、派生成果物、中間物、blob正規配置へ分類する工程が不足したため |
| なぜ次形式でも再発し得るか | 形式ごとのPoC差分に意識が寄り、D2D共通の写像観点でレビューするゲートがなかったため |

## 恒久対策

PoC取り込み時は、修正前に必ず次の5点を1ページ以内で整理する。

| 観点 | 確認内容 | 反映先 |
| --- | --- | --- |
| 1. 能力 | PoCで何を抽出・変換・表示・検証できるか | `srs.md` の能力要求 |
| 2. 内部責務 | データ構造、パーサ、ジェネレータ、UI、検証がどう分かれているか | `sdd_function_architecture.md` |
| 3. データ写像 | PoC出力がD2Dの正本、派生成果物、中間物、blobのどれか | `sdd_data_structure.md` / `sdd_directory.md` |
| 4. UI写像 | PoC UIをWorkbench、Resource、Editor、Command、Eventへどう写すか | `sdd_ui_design.md` |
| 5. 採用判断 | PoC固有のライブラリ、API、ディレクトリ構造を採用・置換・不採用のどれにするか | `sdd_tech_stack.md` |

## 必須ゲート

PoCごとに、以下を満たすまで完了にしない。

- SRSには実装名ではなく、利用者・実装者が理解できる能力名で書く。
- SDDには、PoCの内部責務をD2D境界へ写像した節を追加する。
- データ保存先は、正本、派生成果物、中間物、一時ファイルに分類する。
- UIはPoC画面をそのまま移植せず、Workbench型UXへ写像する。
- 検証観点を最低1つ以上、実装時チェックとして残す。

## 記述ルール

| SRSで書く | SDDで書く |
| --- | --- |
| 文書構造データ、抽出結果、レビュー補助、LLM入力用出力 | `extracted_document.structure_json`、`resource_*`、`source_location`、`blob_resource` |
| 抽出ジョブ、変換ジョブ、レビュー表示 | worker command、JSONL契約、生成ファイル、保存先 |
| 取り込めること、確認できること、切り替えられること | パーサ責務、ジェネレータ責務、UI同期、検証方法 |

## blob分類ルール

| 分類 | 置き場 | 判断基準 |
| --- | --- | --- |
| 抽出中間物 | `blobs/extracted/` | 抽出ジョブ単位の生出力、レビュー補助、再生成可能な中間物 |
| 正規図リソース | `blobs/figures/` | `resource_figure` から参照され、根拠や中間データで再利用される画像 |
| 正規表リソース | `blobs/tables/` | `resource_table` から参照される大容量表データ |
| 派生出力 | `blobs/exports/` または `exports/` | DB to Text、レポート、dump、検索用出力 |

同じファイルが複数分類に見える場合は、「正本から参照される正規配置」を優先し、中間物は再生成可能な補助として扱う。

## 形式別PoCの作業メモ

各PoC取り込み時は、詳細設計に次の節を作る。

- `<形式>抽出ワーカーの出力契約`
- `<形式>抽出ワーカーの内部責務`
- `<形式>抽出で実装時に落とさない観点`

この3節がない場合、PoCの重要知見を設計へ取り込めていない可能性が高い。

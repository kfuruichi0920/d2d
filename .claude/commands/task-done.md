引数 `$ARGUMENTS` をタスクID（例: `T305`、複数の場合はスペース区切り）として扱う。

## 手順

1. **`docs/tasks.md` を読む**

2. **対象タスクの状態を更新**
   - `⏸` または `🔄` → `✅ YYYY-MM-DD`（今日の日付）
   - 複数タスクが指定された場合はすべて更新

3. **フェーズ集計を更新**
   - 該当フェーズの「完了/合計」数を再カウント
   - パーセンテージを更新（小数点以下切捨て）

4. **トータル完了数を更新**
   - ヘッダーの総完了数とパーセンテージを更新
   - 「N回目更新」のNをインクリメント

5. **変更を表示して確認を求める（コミットはしない）**
   - 何が変わったかを一言で伝え、コミットするかどうかをユーザーに確認

## フォーマット例

```markdown
| T305 | 用語集自動抽出 | ✅ 2026-06-28 | LLMを使って中間データから用語候補を抽出 |
```

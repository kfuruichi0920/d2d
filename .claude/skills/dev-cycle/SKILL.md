# dev-cycle

`dev-process/PROCESS.md` に定義された 7 ステップの開発サイクルを、指示されたフェーズ
（または個別タスク）に対して実行する。

手順:

1. `dev-process/STATE.md` を読む（現在地・恒久制約・E2E の注意・残課題）。
2. `dev-process/PROCESS.md` を読み、ステップ 1〜7 を**順番どおり**実行する。
3. 品質ゲートは `node dev-process/verify.mjs --e2e` で確認する（全 PASS が完了条件）。
4. 完了時は STATE.md を更新してコミットし、PROCESS.md の「品質基準（Done の定義）」
   チェックリストを満たしたことを報告に明記する。

引数 `$ARGUMENTS` にフェーズ名やタスク ID が渡された場合はそれを対象とする。
指定がなければ STATE.md の「残課題」から次の推奨フェーズを提案し、ユーザーに確認する。

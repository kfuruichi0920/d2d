# 中間データ処理プラグインソケット設計書

## 1. 位置づけ

本ドキュメントは、中間データ処理プラグインソケットに接続されるプラグインプラグの設計要求を定義する。

中間データ処理プラグインソケットは、②抽出データを入力し、成果物単位に統合された③中間データを生成・更新する。③中間データは、原本に近い自然言語本文、章構成、図表、参照関係、設計観点別情報を含む統合設計書データである。

## 2. 対象プラグインソケット

| 項目 | 内容 |
| --- | --- |
| プラグインソケット種別 | 中間データ処理プラグインソケット |
| 主入力 | ②抽出データ、成果物構成定義、既存③中間データ |
| 主出力 | ③中間データ成果物フォルダ |
| 成果物単位 | 複数原本ファイルから成る成果物単位 |
| 正本 | JSON / JSONL、SQLite DB、統合構造 |
| 派生成果物 | 統合設計書Markdown、LLM入力用チャンク、レビュー用テキスト |

## 3. 責務範囲

| 区分 | 内容 |
| --- | --- |
| 責務 | ②抽出データの取込、成果物単位の統合、章構成整理、自然言語本文の統合、参照リンク生成 |
| 責務 | 図、表、モデル、シナリオ、IF、状態遷移、検証等の設計観点別情報を③中間データとして整理する |
| 責務 | ③中間データからLLM入力用チャンクを作成・編集・削除できるようにする |
| 責務 | 必要に応じて設計編集プラグインソケットの編集機能を呼び出す |
| 非責務 | 文書抽出プラグインソケットの実装へ依存すること |
| 非責務 | LLM候補を確定情報として直接保存すること |
| 非責務 | 履歴差分やトレース分析の正本を直接更新すること |

## 4. 入出力

### 4.1 入力

| 入力 | 内容 |
| --- | --- |
| extracted_artifacts | ②抽出データ成果物フォルダ群 |
| artifact_definition | 成果物ID、成果物名、開発フェーズ、原本ファイル構成、統合順序 |
| existing_intermediate | 編集または再生成対象の既存③中間データ |
| processing_profile | 章構成整理、表正規化、参照リンク生成、チャンク生成の設定 |

### 4.2 出力

| 出力 | 内容 |
| --- | --- |
| manifest | 成果物ID、対象②抽出データ群、schema_version、成果物一覧 |
| integrated_document.md | 原本に近い自然言語本文と章構成を持つ統合設計書 |
| integrated_structure.json | 章節、段落、図表、参照、本文範囲、ID対応 |
| intermediate.sqlite | ③中間データの正本DB |
| chunks.jsonl | LLM入力用の一時的な入力単位 |
| review.md | 中間データレビュー用Markdown |
| logs/ | 処理ログ、警告、統合できなかった情報 |

## 5. 成果物フォルダ

```text
artifacts/intermediate/{design_artifact_id}/
  manifest.json
  integrated_document.md
  integrated_structure.json
  intermediate.sqlite
  chunks.jsonl
  review.md
  media/
  logs/
```

## 6. manifest項目

| 項目 | 内容 |
| --- | --- |
| plugin_socket_type | `intermediate_processor` |
| plugin_id | 中間データ処理プラグインプラグID |
| design_artifact_id | ③中間データ成果物ID |
| source_extracted_ids | 入力にした②抽出データID群 |
| schema_version | ③中間データschema version |
| reads | `artifacts/extracted/*`、既存③中間データ |
| writes | `artifacts/intermediate/{design_artifact_id}` |
| optional_sockets | 設計編集、LLMプロバイダ、履歴管理、トレーサビリティ |

## 7. 設計編集連携

中間データ処理プラグインソケットは、③中間データを整理する過程で、対象箇所の設計観点に応じて設計編集プラグインソケットを呼び出せる。

| 対象 | 呼び出す編集機能 |
| --- | --- |
| 図 | 図編集 |
| 表 | 表編集、構造化表編集 |
| モデル記述 | モデル編集 |
| シナリオ | シナリオ編集 |
| IF | IF編集 |
| 状態遷移 | 状態遷移編集 |

## 8. イベント

| イベント | 方向 | 内容 |
| --- | --- | --- |
| extraction.completed | 購読 | ②抽出データ生成完了 |
| intermediate.started | 発行 | ③中間データ処理開始 |
| intermediate.updated | 発行 | ③中間データ更新 |
| intermediate.confirmed | 発行 | 編集確定済みの③中間データ保存 |
| artifact.updated | 発行 | 成果物フォルダ更新 |

## 9. 編集状態

中間データ処理では、編集中と編集確定を分ける。LLM候補を利用する場合も、LLM候補、UI取込、編集中、UI確定、③中間データ正本反映の順に扱う。

## 10. LLM利用

LLMは、用語抽出候補、参照リンク候補、図表説明候補、設計要素候補、チャンク候補の生成に利用できる。LLM入力には、対象③中間データID、本文、図表参照、原本根拠リンクを含める。

## 11. エラー処理

| 状態 | 扱い |
| --- | --- |
| 入力②抽出データ欠落 | 対象成果物IDと欠落IDをログに記録し、処理を停止または部分処理にする |
| 統合順序不明 | ユーザ確認が必要な警告として扱う |
| ID重複 | 新IDを割り当て、元IDとの対応を保持する |
| チャンク生成失敗 | ③中間データ正本は保持し、チャンクのみ再生成可能にする |

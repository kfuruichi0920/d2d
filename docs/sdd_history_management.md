# 履歴管理機能設計書

## 1. 位置づけ

本ドキュメントは、履歴管理機能の設計を定義する。

履歴管理機能は、②抽出データ、③中間データ、④設計モデル、SQLite DB、ZIPアーカイブ、DB to Text出力を対象に、変更前、特定時点、時点間差分を随時参照できる共通機能である。レポート作成機能は含めない。

## 2. 対象機能

| 項目 | 内容 |
| --- | --- |
| 機能種別 | 履歴管理機能 |
| 主入力 | ②抽出データ、③中間データ、④設計モデル、SQLite DB、ZIPアーカイブ |
| 主出力 | DB差分結果、成果物差分結果、履歴差分ビュー |
| 正本更新 | 行わない |
| 主用途 | 設計編集中、トレース分析中、抽出・中間処理中の差分参照 |

## 3. 責務範囲

| 区分 | 内容 |
| --- | --- |
| 責務 | SQLite DB内容比較、SQLite DB履歴差分確認、②/③/④成果物差分確認を行う |
| 責務 | ZIPアーカイブを差分比較用にインポートし、現在成果物と比較する |
| 責務 | DB to Text出力を利用して差分表示やLLM入力に使える形式へ変換する |
| 責務 | 変更前、特定時点、時点間差分を他機能から参照可能にする |
| 非責務 | 成果物フォルダ、SQLite DB、④設計モデルの正本を直接上書きすること |
| 非責務 | Gitコミットを実行すること |
| 非責務 | レポート作成機能を提供すること |

## 4. 入出力

### 4.1 入力

| 入力 | 内容 |
| --- | --- |
| current_artifact | 現在の②/③/④成果物フォルダまたはDB |
| baseline_artifact | 比較元の成果物フォルダ、ZIPアーカイブ、DB to Text出力 |
| diff_profile | 比較対象、除外項目、ID単位、表示形式 |
| query | 特定時点、対象ID、対象成果物、対象DBテーブル |

### 4.2 出力

| 出力 | 内容 |
| --- | --- |
| diff_summary | 追加、変更、削除、移動、分割、マージの概要 |
| db_diff | SQLite DBのテーブル・レコード・カラム差分 |
| artifact_diff | JSON / JSONL、画像参照、manifest、派生成果物の差分 |
| text_diff | DB to TextまたはMarkdownの差分表示 |
| history_view | UI表示用の差分ビュー |

## 5. 機能定義項目

| 項目 | 内容 |
| --- | --- |
| function_type | `history_management` |
| function_id | 履歴管理機能ID |
| supported_targets | extracted、intermediate、design_model、sqlite、zip |
| supported_diff_types | db、json、jsonl、text、manifest、artifact |
| reads | ②抽出データ、③中間データ、④設計モデル、ZIPアーカイブ |
| writes | 差分結果、履歴差分ビュー、一時インポート領域 |
| optional_functions | トレーサビリティ、LLMプロバイダ |

## 6. ZIPアーカイブ差分

ZIPアーカイブは、過去時点の②抽出データ、③中間データ、④設計モデルを保存し、現在成果物との差分確認に利用する。ZIPインポートは現在の正本成果物を直接上書きしない。

```text
archives/
  extracted/
  intermediate/
  design_model/
tmp/
  imported_archive/{archive_id}/
```

## 7. DB to Text連携

DB to Textは、②抽出データ、③中間データ、④設計モデルに含まれるDB内容を、差分表示やLLM入力のために派生的にテキスト化する機能として利用する。DBが正本であり、Textは正本ではない。

## 8. 連携

| 連携先 | 用途 |
| --- | --- |
| 文書抽出機能 | 抽出前後、再抽出前後の差分参照 |
| 中間データ処理機能 | 統合前後、編集確定前後の差分参照 |
| 設計編集機能 | 編集中に変更前、特定時点、差分を参照する |
| トレーサビリティ機能 | 変更IDに対する影響範囲確認 |
| LLMプロバイダ機能 | 差分要約候補、変更影響候補の生成 |

## 9. イベント

| イベント | 方向 | 内容 |
| --- | --- | --- |
| artifact.updated | 購読 | 成果物更新後の差分参照対象更新 |
| design_model.updated | 購読 | ④設計モデル更新後の差分参照対象更新 |
| archive.imported | 発行 | ZIPアーカイブを差分比較用にインポート |
| history.diff.completed | 発行 | 差分計算完了 |
| history.diff.failed | 発行 | 差分計算失敗 |

## 10. UI・CLI

| 操作 | 要求 |
| --- | --- |
| 差分対象選択 | 現在成果物、ZIPアーカイブ、DB、DB to Text出力を選択できること |
| DB差分表示 | テーブル、レコード、カラム単位の差分を確認できること |
| 成果物差分表示 | ②/③/④のID単位で追加、変更、削除を確認できること |
| 履歴参照 | 設計編集中に変更前、特定時点、時点間差分を参照できること |

## 11. エラー処理

| 状態 | 扱い |
| --- | --- |
| ZIP破損 | インポート失敗としてログに理由を記録する |
| schema_version不一致 | 比較可能範囲を表示し、必要に応じてschema migration対象として扱う |
| DB読込失敗 | 正本DBを変更せず、差分計算を中止する |
| 大規模差分 | 分割表示、遅延ロード、対象範囲指定で確認できるようにする |

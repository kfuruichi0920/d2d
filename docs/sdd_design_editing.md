# 設計編集機能設計書

## 1. 位置づけ

本ドキュメントは、設計編集機能の設計を定義する。

設計編集機能は、③中間データおよび④設計モデルを対象に、設計内容の編集、検索、レビュー補助、モデル表現編集を行う。中間データ処理機能から呼び出される場合もある。

## 2. 対象機能

| 項目 | 内容 |
| --- | --- |
| 機能種別 | 設計編集機能 |
| 主入力 | ③中間データ、④設計モデル、用語、レビュー状態 |
| 主出力 | 更新済み③中間データ、④設計要素、関係、レビュー状態、モデル表現 |
| 編集対象 | 自然言語本文、図、表、モデル、シナリオ、IF、状態遷移、検証 |
| 正本反映 | UI確定操作後に反映する |

## 3. 責務範囲

| 区分 | 内容 |
| --- | --- |
| 責務 | ③中間データの本文、図、表、モデル、シナリオ、IF、状態遷移等を編集する |
| 責務 | ④設計モデルの設計要素、関係、レビュー状態、モデル表現を編集する |
| 責務 | 用語検索、根拠リンク表示、履歴差分参照、トレース分析呼び出しを提供する |
| 責務 | PlantUML、Mermaid、SysMLv2等のモデル表現編集を扱う |
| 非責務 | ①原本データを改変すること |
| 非責務 | ②抽出データを直接再抽出すること |
| 非責務 | LLM候補をレビューなしで確定すること |

## 4. 編集機能

| 機能 | 内容 |
| --- | --- |
| 用語検索 | ③中間データ、④設計モデル、用語辞書から検索する |
| 図編集 | 図、キャプション、説明文、図内要素、図表番号を編集する |
| 表編集 | 表、行、列、セル、表構造、セル根拠を編集する |
| モデル編集 | 構造、要素、関係、モデル表現を編集する |
| シナリオ編集 | 手順、イベント、アクション、期待結果を編集する |
| IF編集 | API、通信、信号、入出力、データ項目を編集する |
| 状態遷移編集 | 状態、遷移、イベント、ガード条件、アクションを編集する |

## 5. 入出力

### 5.1 入力

| 入力 | 内容 |
| --- | --- |
| target_data | ③中間データIDまたは④設計モデルID |
| edit_context | 対象章節、設計観点、表示範囲、レビュー状態 |
| candidate_data | LLM候補、ルールベース候補、差分候補 |
| reference_data | 原本根拠リンク、履歴差分、トレース関係 |

### 5.2 出力

| 出力 | 内容 |
| --- | --- |
| edit_session | 編集中データ、差分、ロック状態 |
| confirmed_update | UI確定後に保存する③中間データまたは④設計モデル更新 |
| review_update | レビュー状態、採否履歴、コメント |
| model_representation | PlantUML、Mermaid、SysMLv2等の派生表現 |

## 6. 機能定義項目

| 項目 | 内容 |
| --- | --- |
| function_type | `design_editing` |
| function_id | 設計編集機能ID |
| editor_type | term、diagram、table、model、scenario、if、state_machine等 |
| supported_targets | intermediate、design_model |
| reads | ③中間データ、④設計モデル、用語、履歴、トレース |
| writes | ③中間データ、④設計モデル、レビュー状態、派生成果物 |
| optional_functions | LLMプロバイダ、履歴管理、トレーサビリティ |

## 7. 編集状態

| 状態 | 内容 |
| --- | --- |
| 候補 | LLMまたはルールベースで生成された未取込候補 |
| 編集中 | UIに取り込まれ、人間が編集中の状態 |
| 編集確定 | UI確定操作により③または④の正本へ反映する状態 |
| 棄却 | 候補または編集中内容を採用しない状態 |

LLM候補を利用する場合は、LLM候補、UI取込操作、編集中、UI確定操作、正本反映の順に扱う。

## 8. 連携

| 連携先 | 用途 |
| --- | --- |
| 中間データ処理機能 | ③中間データ整理中の編集機能呼び出し |
| LLMプロバイダ機能 | 編集候補、要約、モデル表現候補の生成 |
| 履歴管理機能 | 変更前、特定時点、時点間差分の参照 |
| トレーサビリティ機能 | 対象IDの関係確認、影響分析、カバレッジ確認 |

## 9. イベント

| イベント | 方向 | 内容 |
| --- | --- | --- |
| edit.session.started | 発行 | 編集開始 |
| edit.candidate.imported | 発行 | 候補をUIへ取り込み |
| edit.confirmed | 発行 | 編集確定 |
| intermediate.updated | 発行 | ③中間データ更新 |
| design_model.updated | 発行 | ④設計モデル更新 |
| relation.updated | 発行 | 関係更新 |

## 10. UI・CLI

| 操作 | 要求 |
| --- | --- |
| 編集 | 対象ID、根拠、現在値、候補、履歴差分を見ながら編集できること |
| 確定 | 編集中内容を確定し、③または④の正本へ保存できること |
| 差分確認 | 編集前後、ZIPアーカイブ、DB to Text出力との差分を確認できること |
| トレース確認 | 編集対象IDの上流・下流関係を確認できること |

## 11. エラー処理

| 状態 | 扱い |
| --- | --- |
| 競合 | 編集対象の版差異を表示し、上書き前に確認する |
| ID不整合 | 保存前に検出し、修正または保存中止できるようにする |
| モデル表現エラー | 派生表現のみエラー扱いとし、正本データを破棄しない |
| LLM候補不正 | 候補を未採用状態で保持し、検証エラーを表示する |

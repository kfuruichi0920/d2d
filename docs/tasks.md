## 管理ルール

1. **着手時**: ステータスを `🔄 進行中` に変更し、計画日欄に着手日を記入する。
2. **完了時**: ステータスを `✅ 完了` に変更し、実績欄に完了日（YYYY-MM-DD）を記入する。
3. **タスク追加**: Phase 末尾に行を追加し、連番で採番する（例: T209 の次は T210）。
4. **進捗サマリー**: タスク変更のたびにサマリー行の数字を更新する。
5. **保留**: 仕様変更・優先度変更等で延期するタスクは `⏸` にしてその理由をタスク行のコメント列（備考）に追記する。

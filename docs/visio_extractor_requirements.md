# Visio文書情報抽出プラグイン 詳細要求

## 1. 位置付け

本ドキュメントは、D2D(設計情報デジタル化・トレーサビリティ支援ツール)における Visio 文書情報抽出機能の詳細要求を定義する。

Visio 文書抽出機能は、`docs/srs_main.md` の以下に対応する詳細要求である。

- `3.1 対象文書` の Visio 文書取込要求
- `7.1 原本取込` の Visio 文書取込要求
- `7.2 文書構造抽出` の図形、接続、ラベル情報抽出要求
- `7.3 抽出結果レビュー` の原本表示と抽出結果表示の並列確認要求
- `8. 中間データ処理要求` のチャンク化、図表説明、参照リンク生成要求
- `10. トレース要求` の関係グラフ、影響分析、トレース探索要求

本機能は単独の Visio 変換ツールではなく、D2D のプラグイン構成、ジョブ管理、ZIP パッケージ管理、設計情報ストア管理、LLM Gateway、レビュー UI と連携する Visio 取込・抽出プラグインとして実装する。

本ドキュメントは、[文書抽出プラグイン 共通設計要求](extractor_common_design_requirements.md) を前提とする形式別要求である。

ここに記載する XML パート名、内部ファイル名、ライブラリ名、UI 構成、処理アルゴリズムは、相互運用や検証のための参考情報であり、明示的に「必須」と書かれていない限り実装方式を固定しない。

## 2. 基本方針

| ID | 要求 |
| --- | --- |
| VDEX-001 | Visio 抽出機能は `plugin.visio-importer` または同等の Visio 取込プラグインとして提供すること。 |
| VDEX-002 | 抽出処理、プレビュー生成、LLM 補助処理は UI スレッドから分離し、D2D のジョブ管理機構から実行できること。 |
| VDEX-003 | 原本 `.vsdx` / `.vsdm` は改変せず、原本ハッシュ、ファイル名、取込日時、抽出器バージョンを保持すること。 |
| VDEX-004 | 抽出結果は、原本忠実な抽出データと、レビュー・LLM 入力向けに整理した中間データを分離して保持すること。 |
| VDEX-005 | ページ、図形、コネクタ、テキスト、画像、レイヤー、グループ、接続関係は必須抽出観点として扱うこと。 |
| VDEX-006 | 図形が要求、機能、状態、処理、判断、IF などを表すかどうかは候補として生成し、人間レビューで確定できること。 |
| VDEX-007 | LLM を利用する場合でも、ページ構造、図形 ID、接続情報、座標、成果物保存、根拠リンク作成はルールベースで実施すること。 |
| VDEX-008 | LLM 出力は候補として保持し、抽出データそのものを上書きしないこと。 |

## 3. 対象ファイル

| ID | 要求 |
| --- | --- |
| VDEX-FILE-001 | `.vsdx` を対象にできること。 |
| VDEX-FILE-002 | `.vsdm` はマクロを実行せず、文書構造抽出対象として扱えること。 |
| VDEX-FILE-003 | `.vsd` 等の旧バイナリ形式は初期スコープ外とし、変換済みファイルまたは別プラグインで扱える余地を残すこと。 |
| VDEX-FILE-004 | パスワード保護、暗号化、破損、不正なファイルは抽出失敗として扱い、ジョブログに理由を記録すること。 |
| VDEX-FILE-005 | ページ数、図形数、コネクタ数、画像数、レイヤー数など、処理負荷の事前見積りに必要な情報をプリチェックで取得できること。 |

## 4. 抽出データモデル

Visio 抽出結果は、文書単位のメタデータ、ページ、図形、コネクタ、グループ、テキスト、画像、レイヤー、接続関係、抽出ログに分けて保持する。

| ID | 要求 |
| --- | --- |
| VDEX-MODEL-001 | 抽出データは JSON / JSONL として保存できること。 |
| VDEX-MODEL-002 | 抽出データは ZIP パッケージ内に格納できること。 |
| VDEX-MODEL-003 | 抽出データの schema_version を保持すること。 |
| VDEX-MODEL-004 | 図形、コネクタ、ページ、画像、レイヤーに安定 ID を付与できること。 |
| VDEX-MODEL-005 | 原本内 ID と D2D 内部 ID の対応を保持できること。 |
| VDEX-MODEL-006 | 抽出時に確定できない情報は空文字で埋めず、未取得、対象外、推定、抽出失敗、未対応を区別できる状態値で表すこと。 |
| VDEX-MODEL-007 | Markdown 出力は閲覧・LLM 入力用の派生成果物とし、正本はページ中心の構造化 JSON / JSONL とすること。 |

## 5. Visio パッケージ解析要求

| ID | 要求 |
| --- | --- |
| VDEX-PKG-001 | `.vsdx` / `.vsdm` を ZIP パッケージとして読み取り、Visio XML パートを解析できること。 |
| VDEX-PKG-002 | `visio/document.xml`、`visio/pages/pageN.xml`、`visio/masters/`、relationship、docProps から取得可能な情報を解析できること。 |
| VDEX-PKG-003 | ページ、マスター、スタイル、テーマ、画像、埋め込みオブジェクトへの relationship を解決できること。 |
| VDEX-PKG-004 | パート欠落、不正 XML、未対応要素があっても、文書全体を即時破棄せず、可能な範囲の抽出結果と警告を返すこと。 |

| VDEX-SHAPE-005 | 図形内テキスト、複数行テキスト、フィールド由来テキストを可能な範囲で抽出できること。 |

## 8. コネクタ・接続関係抽出要求

| ID | 要求 |
| --- | --- |
| VDEX-CONN-001 | コネクタ図形、始点、終点、接続先図形候補を抽出できること。 |
| VDEX-CONN-002 | 矢印向き、線種、ラベル、接続点を可能な範囲で保持できること。 |
| VDEX-CONN-003 | 接続情報は relation 候補として保存し、確定前、確定済、棄却を区別できること。 |
| VDEX-CONN-004 | 接続方向が曖昧な場合は推定値と警告を分けて記録すること。 |
| VDEX-CONN-005 | 確定した接続関係は、設計モデルの関係性または GraphDB / 関係グラフ索引へ反映できること。 |

## 9. 画像・埋め込みオブジェクト抽出要求

| ID | 要求 |
| --- | --- |
| VDEX-MEDIA-001 | Visio 文書内の画像を抽出し、ZIP パッケージ内の `media/` 相当領域に保存できること。 |
| VDEX-MEDIA-002 | 画像のページ、図形 ID、bbox、ハッシュ、サイズを保持できること。 |
| VDEX-MEDIA-003 | 埋め込み Office 文書、OLE、外部リンクは未対応要素または参照情報として記録し、原本位置と警告を保持すること。 |
| VDEX-MEDIA-004 | ページ全体の概要画像を生成できる構成とすること。ただしレンダリング結果は派生成果物であり、抽出データの正本ではないこと。 |

| VDEX-OUT-002 | LLM 入力向け Markdown を生成できること。 |

| VDEX-OUT-006 | 接続関係は JSON / JSONL、Markdown、GraphDB 投入用データとして出力できること。 |

## 11. D2D 連携要求

| ID | 要求 |
| --- | --- |
| VDEX-D2D-001 | Visio 抽出ジョブは `CORE-020` から `CORE-024` のジョブ管理要求に従うこと。 |
| VDEX-D2D-002 | 抽出成果物は `DATA-001` から `DATA-007` の ZIP パッケージ管理要求に従うこと。 |
| VDEX-D2D-003 | 抽出済み要素は中間データ処理でページ単位、図形単位、接続関係単位にチャンク化できること。 |
| VDEX-D2D-004 | 抽出済みページ、図形、コネクタ、画像、ラベルは設計モデル要素の `source_links` として参照できること。 |
| VDEX-D2D-005 | 抽出結果レビュー UI で、未確認、確認済、要修正、棄却を管理できること。 |
| VDEX-D2D-006 | Visio ビューでは、ページ一覧、プレビュー、図形オーバーレイ、接続関係の強調表示を提供できること。 |
| VDEX-D2D-007 | 図形・接続候補は D2D の図表説明生成、参照リンク生成、関係グラフ、設計モデル候補生成へ渡せること。 |
| VDEX-D2D-008 | LLM に渡す入力は、抽出済み要素 ID と原本根拠リンクを含む source-grounded な形式にすること。 |
| VDEX-D2D-009 | LLM 送信前に、対象ページ、対象図形、原本根拠、送信予定テキスト、概要画像の有無を確認できること。 |
| VDEX-D2D-010 | 外部 LLM API が無効なワークスペースでは、Visio 抽出とレビューはローカル処理のみで完結できること。 |

## 12. 初期スコープ

初期実装では以下を優先する。

| 優先度 | 内容 |
| --- | --- |
| Must | `.vsdx` 取込、原本ハッシュ、ページ一覧、図形、テキスト、コネクタ、接続候補、画像、JSON / Markdown 出力、ZIP パッケージ保存 |
| Should | マスター参照、レイヤー、グループ、ページ概要画像、GraphDB 投入用関係データ |
| Could | データグラフィック、テーマ詳細、カスタムプロパティ詳細、外部データリンク |
| Later | `.vsd` 直接解析、完全レンダリング、OLE 内容解析、Visio 編集機能 |

## 13. 参考実装から持ち込まない前提

| 項目 | 方針 |
| --- | --- |
| Visio アプリ依存の抽出 | 初期要件では Visio アプリのインストールを前提にしない。必要な場合も外部ワーカー候補として扱う。 |
| レンダリング結果を正本とする運用 | 正本は構造化 JSON / JSONL とし、レンダリング画像はレビュー・LLM 入力・エクスポート用の派生物とする。 |
| 抽出器単体での意味分類確定 | 図形が要求、機能、状態、IF などのどれに該当するかは、中間データ処理、LLM 候補生成、人間レビューで確定する。 |

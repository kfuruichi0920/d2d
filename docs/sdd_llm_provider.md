# LLMプロバイダ機能設計書

## 1. 位置づけ

本ドキュメントは、LLMプロバイダ機能の設計を定義する。

LLMプロバイダ機能は、文書抽出、中間データ処理、設計編集、トレーサビリティ、履歴管理から横断的に呼び出される候補生成機能である。LLM出力は候補であり、②抽出データ、③中間データ、④設計モデルの正本を直接更新しない。

## 2. 対象機能

| 項目 | 内容 |
| --- | --- |
| 機能種別 | LLMプロバイダ機能 |
| 主入力 | source-groundedな入力、プロンプト、モデル設定、外部送信可否 |
| 主出力 | LLM候補、実行ログ、根拠リンク、使用量 |
| 成果物単位 | 呼び出し元ジョブまたは候補生成単位 |
| 正本更新 | 行わない |

## 3. 責務範囲

| 区分 | 内容 |
| --- | --- |
| 責務 | 外部LLM、ローカルLLM、クラウドLLMのプロバイダ接続を抽象化する |
| 責務 | 外部送信可否、ローカル限定、機密情報マスク、モデル設定を制御する |
| 責務 | 候補生成、要約、分類、関係候補生成、説明生成を実行する |
| 責務 | 入力、出力、プロンプト、モデル、実行日時、根拠、採否状態をログ化する |
| 非責務 | 生成候補を正本へ直接反映すること |
| 非責務 | 呼び出し元データの編集確定を代行すること |

## 4. 入出力

### 4.1 入力

| 入力 | 内容 |
| --- | --- |
| request_id | LLM実行単位ID |
| caller_function_type | 呼び出し元機能種別 |
| source_context | ②/③/④の対象ID、本文、図表参照、原本根拠リンク |
| prompt | プロンプト、出力schema、制約 |
| model_profile | モデル名、温度、最大出力、ローカル/外部設定 |
| policy | 外部送信可否、マスク設定、ログ制約 |

### 4.2 出力

| 出力 | 内容 |
| --- | --- |
| candidate_id | LLM候補ID |
| candidate_payload | 構造化候補、本文候補、分類候補、関係候補 |
| evidence_links | 入力に含まれる根拠ID、原本位置 |
| validation_result | JSON Schema検証結果、警告 |
| usage | token数、処理時間、モデル情報 |
| llm_log | 入力要約、プロンプト、応答、マスク状態、エラー |

## 5. 機能定義項目

| 項目 | 内容 |
| --- | --- |
| function_type | `llm_provider` |
| function_id | LLMプロバイダ機能ID |
| provider_type | external、local、cloud、mock |
| supported_models | 対応モデル |
| supports_structured_output | 構造化出力可否 |
| supports_vision | 画像入力可否 |
| external_send | 外部送信有無 |
| secret_keys | 必要な機密設定 |
| license | 利用ライブラリとライセンス |

## 6. 呼び出し元

| 呼び出し元 | 用途 |
| --- | --- |
| 文書抽出 | OCR補正、表候補、図表説明候補 |
| 中間データ処理 | 用語抽出、参照リンク、チャンク候補、設計要素候補 |
| 設計編集 | 編集候補、要約、モデル表現候補、状態遷移候補 |
| トレーサビリティ | 関係候補、影響分析補助、欠落候補 |
| 履歴管理 | 差分要約、変更影響候補 |

## 7. イベント

| イベント | 方向 | 内容 |
| --- | --- | --- |
| llm.requested | 購読 | LLM実行要求 |
| llm.started | 発行 | LLM実行開始 |
| llm.candidate.generated | 発行 | LLM候補生成完了 |
| llm.failed | 発行 | LLM実行失敗 |
| llm.policy.blocked | 発行 | 外部送信禁止等により実行不可 |

## 8. UI・CLI

| 操作 | 要求 |
| --- | --- |
| 送信前確認 | 対象ID、根拠、送信予定本文、画像有無、外部送信先を確認できること |
| 候補確認 | 候補、根拠、検証結果、採用・修正・棄却状態を確認できること |
| 設定 | プロバイダ、モデル、ローカル限定、外部送信可否を設定できること |
| ログ確認 | 機密情報を除いたLLM実行ログを確認できること |

## 9. セキュリティ

| 項目 | 要求 |
| --- | --- |
| 外部送信制御 | `feature.llm.externalApiAllowed` とワークスペース設定に従う |
| ローカル限定 | `feature.llm.localOnly` が有効な場合は外部LLMを呼び出さない |
| 機密情報 | APIキー、認証情報、未マスク機密本文をログに保存しない |
| 入力制限 | LLMへ渡す入力は必要な範囲の本文、画像、ID、根拠リンクに制限する |

## 10. エラー処理

| 状態 | 扱い |
| --- | --- |
| 外部送信不可 | 候補生成を実行せず、呼び出し元にポリシーブロックを返す |
| JSON Schema不一致 | 候補を未採用状態で保存し、検証エラーを表示する |
| タイムアウト | 再実行条件をログに記録する |
| プロバイダ障害 | 呼び出し元がルールベース候補または手動編集へ戻れるようにする |

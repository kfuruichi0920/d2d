# トレーサビリティプラグインソケット設計書

## 1. 位置づけ

本ドキュメントは、トレーサビリティプラグインソケットに接続されるプラグインプラグの設計要求を定義する。

トレーサビリティプラグインソケットは、②抽出データ、③中間データ、④設計モデルのすべての情報単位IDに対して、関係管理、検索、影響分析、カバレッジ分析を提供する共通機能である。

## 2. 対象プラグインソケット

| 項目 | 内容 |
| --- | --- |
| プラグインソケット種別 | トレーサビリティプラグインソケット |
| 主入力 | ②抽出データID、③中間データID、④設計モデルID、関係データ |
| 主出力 | トレースマトリクス、関係クエリ結果、影響分析結果、カバレッジ分析結果 |
| 正本更新 | 関係データを更新する。②/③/④の正本内容は直接編集しない |

## 3. 責務範囲

| 区分 | 内容 |
| --- | --- |
| 責務 | 情報単位ID間の関係を登録、編集、削除、検索する |
| 責務 | parent_child、based_on、satisfy、verify、dependを基本関係種別として扱う |
| 責務 | トレースマトリクス、関係グラフ、階層リスト、CLIクエリを提供する |
| 責務 | 影響分析、カバレッジ分析、孤立要素検出、欠落検出を行う |
| 非責務 | ②抽出データ、③中間データ、④設計モデルの本文や属性を直接編集すること |
| 非責務 | LLM候補を確定関係として直接登録すること |

## 4. 入出力

### 4.1 入力

| 入力 | 内容 |
| --- | --- |
| source_ids | 関係元ID群 |
| target_ids | 関係先ID群 |
| relation_data | 関係種別、方向、根拠、粒度、レビュー状態 |
| query | 検索条件、分析条件、表示条件 |

### 4.2 出力

| 出力 | 内容 |
| --- | --- |
| relation_records | 関係ID、関係元、関係先、関係種別、根拠リンク |
| trace_matrix | 行・列・関係種別で表現したマトリクス |
| impact_result | 変更影響範囲、上流・下流要素、根拠 |
| coverage_result | 要求・設計・検証等の対応状況 |
| relation_graph | グラフ表示またはGraphDB投入用データ |

## 5. 関係データ

| 属性 | 内容 |
| --- | --- |
| relation_id | 一意識別子 |
| relation_type | parent_child、based_on、satisfy、verify、depend等 |
| direction | 無方向、左から右、右から左、双方向 |
| from_ids | 関係元ID群 |
| to_ids | 関係先ID群 |
| evidence_links | ②/③/④の根拠ID、原本位置 |
| granularity | 成果物、章節、段落、図表、要素、セル等 |
| review_status | 未確認、確認済、要修正、棄却 |
| created_by | 人間、LLM、ルール |

## 6. manifest項目

| 項目 | 内容 |
| --- | --- |
| plugin_socket_type | `traceability` |
| plugin_id | トレーサビリティプラグインプラグID |
| supported_relation_types | 対応関係種別 |
| supported_views | matrix、graph、tree、list、cli |
| reads | ②抽出データ、③中間データ、④設計モデル、関係DB |
| writes | 関係DB、関係索引、分析結果 |
| optional_sockets | LLMプロバイダ、履歴管理 |

## 7. 連携

| 連携先 | 用途 |
| --- | --- |
| 文書抽出プラグインソケット | ②抽出データIDを参照する |
| 中間データ処理プラグインソケット | ③中間データID、成果物ID、章節IDを参照する |
| 設計編集プラグインソケット | 編集対象IDの関係確認、影響分析を提供する |
| LLMプロバイダプラグインソケット | 関係候補、欠落候補、影響説明候補を生成する |
| 履歴管理プラグインソケット | 関係変更前後、時点間差分を参照する |

## 8. イベント

| イベント | 方向 | 内容 |
| --- | --- | --- |
| artifact.updated | 購読 | ②/③成果物更新に伴うID索引更新 |
| design_model.updated | 購読 | ④設計モデル更新に伴う関係検証 |
| relation.updated | 発行 | 関係登録・編集・削除 |
| trace.analysis.completed | 発行 | トレース分析完了 |
| trace.inconsistency.detected | 発行 | 存在しないID、重複、循環、方向不整合の検出 |

## 9. UI・CLI

| 操作 | 要求 |
| --- | --- |
| 関係表示 | マトリクス、グラフ、階層リストで表示できること |
| 関係編集 | IDを指定して関係を登録、修正、削除できること |
| 影響分析 | 任意IDから上流・下流の影響範囲を確認できること |
| カバレッジ分析 | 要求、設計、検証等の対応有無を確認できること |
| CLIクエリ | 条件指定で関係検索と分析を実行できること |

## 10. LLM利用

LLMは関係候補、欠落候補、影響説明候補の生成に利用できる。LLM候補は未確認状態で保持し、人間レビューにより採用、修正、棄却する。

## 11. エラー処理

| 状態 | 扱い |
| --- | --- |
| 存在しないID参照 | 不整合として検出し、関係保存を中止または警告状態にする |
| 循環関係 | 関係種別ごとの許容可否に従い、警告またはエラーとする |
| 重複関係 | 重複候補として表示し、統合または棄却できるようにする |
| 粒度不一致 | 成果物、章節、要素等の粒度を表示し、レビュー対象にする |
